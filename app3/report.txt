App 3 was implemented using an array of circular doubly linked lists, with the indicies of the array acting as ages 18-30 and the nodes in each list containing the ID and age of the students (who are inserted as they are generated). This allows for O(1) inserations, deletions, and mex age gap calulations and O(m) age searches (where m is the number of nodes in a specific list in the array). This implementation is superior to a standard linked list as it allows for O(1) max age gap calculation and slighlty optimized age searches compared to linear time. This is likewise much faster than an array which would have to rely on logarithmic time binary search and shifting to maintain sortedness, or a hash table which could have O(n) insertions in the worst case when accounting for linear probing.

Function Time Complexities:

createDataSet: O(n)

destroyDataSet: O(n) (where n = number of nodes in the dataset)

searchAge: O(m) (where m = number of nodes in a specific LL in the array)

searchID: O(n)

insertion: O(1)

deletion: O(1)

maxAgeGap: O(1) (O(13) but simplifies to O(1) constant time)
